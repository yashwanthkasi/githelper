Scenario 1: Your First Day - The Clone Wars

Beginner

"Welcome to the team! Here's the repo URL. Get the code running locally."

üé¨ The Situation

It's your first day at Infosys. Your team lead sends you a message: "Hey! Here's our project repository. Clone it and set up your local environment. Let me know if you need help!"

üìù Step-by-Step Solution

1
Get the Repository URL

Your team lead will share a URL that looks like:

Repository URL 
https://github.com/infosys-team/project-name.git
2
Open Terminal & Navigate to Your Workspace

Open Git Bash or Command Prompt and go to where you want the project:

Terminal 
cd C:\Users\YourName\Projects
3
Clone the Repository

This downloads the entire project to your computer:

Terminal 
git clone https://github.com/infosys-team/project-name.git
Output: 
Cloning into 'project-name'...
remote: Enumerating objects: 1250, done.
remote: Counting objects: 100% (1250/1250), done.
Receiving objects: 100% (1250/1250), 2.5 MiB | 1.2 MiB/s, done.
Resolving deltas: 100% (820/820), done.
4
Enter the Project Folder
Terminal 
cd project-name
5
Verify Everything is Working

Check the status to confirm Git is tracking the project:

Terminal 
git status
Output: 
On branch main
Your branch is up to date with 'origin/main'.

nothing to commit, working tree clean
üí°
Beginner Tip: Think of git clone like downloading a ZIP file, but smarter! It also downloads the entire history of changes, so you can see who did what and when.

Beginner

1

Terminal

2

Terminal

3
Verify Your Settings
Terminal

Important: Use your official Infosys email for work projects. This ensures proper attribution in code reviews and audits.

Beginner

üé¨ The Situation

Scenario 4: Feature Branch Friday

Intermediate

"PM says: Build the login page. Don't touch the main branch!"

üé¨ The Situation

Your PM assigns you a new feature: build a login page. The rule is simple - never work directly on the main branch. Create a feature branch, do your work there, then merge it back when done!

üìù Step-by-Step Solution

1
Make Sure You're on Main & Up to Date
Terminal 
git checkout main
git pull
2
Create & Switch to a New Branch
Terminal 
git checkout -b feature/login-page
Output: 
Switched to a new branch 'feature/login-page'
3
Do Your Work & Commit

Write your code, then save your progress:

Terminal 
git add .
git commit -m "Add login page UI"
4
Push Your Branch to Remote
Terminal 
git push -u origin feature/login-page
üí°
The -u flag links your local branch to the remote. After this, you can just use git push!
5
Create a Pull Request (PR)

Go to GitHub/GitLab/Bitbucket and create a Pull Request to merge your branch into main. Your team will review your code!

6
After PR is Approved - Merge Locally (Alternative)
Terminal 
git checkout main
git merge feature/login-page

üìõ Branch Naming Conventions

feature/ New features ‚Üí feature/user-login
bugfix/ Bug fixes ‚Üí bugfix/login-crash
hotfix/ Urgent production fixes ‚Üí hotfix/security-patch

‚úÖ Commands You Learned

git branch git checkout git checkout -b git merge git push -u

Scenario 5: Context Switching

Intermediate

"Urgent bug! Park your current work and switch to fix it."

üé¨ The Situation

You're in the middle of building a feature when suddenly - URGENT BUG! Your manager needs you to fix it immediately. But you don't want to lose your current work...

üìù Step-by-Step Solution

1
Stash Your Current Work

This saves your uncommitted changes temporarily:

Terminal 
git stash save "WIP: login page styling"
2
Switch to Main & Create Bugfix Branch
Terminal 
git checkout main
git pull
git checkout -b bugfix/urgent-crash
3
Fix the Bug, Commit & Push
Terminal 
git add .
git commit -m "Fix critical crash on homepage"
git push -u origin bugfix/urgent-crash
4
Go Back to Your Feature & Restore Work
Terminal 
git checkout feature/login-page
git stash pop
üí°
Stash Commands:
git stash list - See all stashed changes
git stash pop - Restore and remove from stash
git stash apply - Restore but keep in stash

‚úÖ Commands You Learned

git stash git stash pop git stash list

Scenario 6: The Merge Conflict Monster

Advanced

"You and your colleague both edited the same file. Git is confused!"

üé¨ The Situation

You and your teammate both changed the same line in a file. When you try to merge or pull, Git doesn't know which version to keep. Don't panic - this is normal and easy to fix!

üîç Anatomy of a Conflict

config.js (with conflict markers) 
const API_URL = 
<<<<<<< HEAD
  "https://api.production.com"
=======
  "https://api.staging.com"
>>>>>>> feature/update-api
<<<<<<< HEAD Your current changes
======= Divider between versions
>>>>>>> branch-name Incoming changes

üìù Step-by-Step Solution

1
Don't Panic! See Which Files Have Conflicts
Terminal 
git status
Output: 
Unmerged paths:
  both modified:   config.js
2
Open the File & Resolve

Edit the file and keep what you want. Remove the conflict markers:

config.js (resolved) 
const API_URL = "https://api.production.com"
3
Mark as Resolved & Commit
Terminal 
git add config.js
git commit -m "Resolve merge conflict in config.js"
4
Abort Merge (If Things Go Wrong)

If you want to start over:

Terminal 
git merge --abort
‚ö†Ô∏è
Pro Tip: Talk to your teammate before resolving! Make sure you don't accidentally delete their important changes. Communication prevents conflicts from becoming problems!

‚úÖ Commands You Learned

Conflict resolution git merge --abort

Scenario 7: The Accidental Commit

Intermediate

"Oh no! I committed the wrong file / wrong message / too early!"

üé¨ The Situation

You made a commit but something's wrong - maybe you included a file by mistake, the message has a typo, or you want to add more changes. Don't worry, Git has your back!

üîÑ Common Undo Scenarios

Case 1: Fix the Last Commit Message
Terminal 
git commit --amend -m "New correct message"
Case 2: Add Forgotten Files to Last Commit
Terminal 
git add forgotten-file.js
git commit --amend --no-edit
Case 3: Undo Last Commit (Keep Changes)
Terminal 
git reset --soft HEAD~1

Your files stay modified, commit is undone. Safe!

Case 4: Completely Discard Last Commit
Terminal 
git reset --hard HEAD~1
‚ö†Ô∏è
Danger! This deletes your changes permanently!
Case 5: Discard Uncommitted Changes in a File
Terminal 
git checkout -- filename.js
Case 6: Undo a Pushed Commit (Safe Way)
Terminal 
git revert HEAD

Creates a new commit that undoes the previous one. Safe for shared branches!

Case 7: Unstage a Staged File
Terminal 
git restore --staged filename.js
üí°
Rule of Thumb: If you haven't pushed yet, you can safely amend or reset. Once pushed, use git revert instead to create a new commit that undoes the change (safer for team collaboration).

‚úÖ Commands You Learned

git commit --amend git reset git revert git restore
