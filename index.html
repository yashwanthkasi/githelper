1.1 What is Git & Why You Need It

Beginner

Understanding version control and why Git is essential for every developer

üéØ What is Git?

Git is a distributed version control system that tracks changes in your code over time. Think of it like a time machine for your projects‚Äîyou can see what changed, when it changed, who changed it, and why.

‚ùå Without Git
project_final.zip project_final_v2.zip project_REAL_final.zip project_USE_THIS_ONE.zip project_FINAL_FINAL.zip
‚úÖ With Git
üìÅ infy-employee-portal/ ‚îî‚îÄ‚îÄ Complete history tracked ‚îî‚îÄ‚îÄ Every change recorded ‚îî‚îÄ‚îÄ Easy collaboration

üöÄ Why Git Matters

  • Seamless collaboration - Multiple developers work on the same codebase without conflicts
  • Complete history - Every change is tracked with who, what, when, and why
  • Safe experimentation - Try new features without breaking working code
  • Easy recovery - Revert mistakes or recover deleted work in seconds
  • Industry standard - Used by millions of developers and every tech company including Infosys

üîÑ Git vs GitHub - They're Not the Same!

A common beginner confusion: Git and GitHub are different things.

Git (Local)

Version control tool that runs on your computer

üñ•Ô∏è Like Microsoft Word (the software)

‚Üí
GitHub (Remote)

Cloud platform that hosts your Git repositories online

‚òÅÔ∏è Like Google Docs (the online platform)

üí°
Alternatives to GitHub: GitLab, Bitbucket, Azure DevOps - Infosys uses different platforms for different clients.

2.1 Create a Project from Scratch

Beginner

Start a new project with Git and push it to GitHub - the complete flow

üé¨ The Situation

You've been assigned to create a new microservice for the employee portal. You need to set up a fresh Git repository and push it to your team's GitHub.

üìù Complete Flow: Local ‚Üí GitHub

1
Create Project Folder
Terminal

cd infy-employee-portal
2
Initialize Git Repository
Terminal 
git init
Output: 
Initialized empty Git repository in .../infy-employee-portal/.git/
This creates a hidden .git folder - Git's brain that stores all tracking data.
3
Create .gitignore (Important!)

Before adding files, set up .gitignore to exclude files that shouldn't be tracked:

.gitignore 
# Dependencies
node_modules/
vendor/

# Environment files (NEVER commit these!)
.env
.env.local
*.pem

# Build outputs
dist/
build/
*.class

# IDE settings
.idea/
.vscode/
*.swp

# Logs
*.log
logs/

# OS files
.DS_Store
Thumbs.db
‚ö†Ô∏è
Never commit API keys, passwords, or .env files! This is a security violation.
4
Create Initial Files
Terminal 
echo "# Infy Employee Portal" > README.md
echo "console.log('Hello Infosys!');" > index.js
5
Stage and Commit
Terminal 
git add .
git commit -m "INF-1001: Initial project setup"
6
Create GitHub Repository
  1. Go to github.com/new
  2. Name it: infy-employee-portal
  3. Don't initialize with README (we already have one)
  4. Click "Create repository"
7
Connect Local to Remote & Push
Terminal 
git remote add origin https://github.com/your-username/infy-employee-portal.git
git branch -M main
git push -u origin main
-u sets upstream so future git push works without specifying remote/branch.
‚úÖ
Done! Your code is now on GitHub. Refresh the page to see your files.

4.2 Finding Who Changed What (git blame)

Intermediate

Investigate when a bug was introduced and by whom

üé¨ The Situation

There's a bug in UserService.java line 42. You need to find out when this line was changed and who changed it to understand the context.

üìù Investigation Steps

1
See Who Wrote Each Line
Terminal 
git blame src/UserService.java
Output: 
abc1234 (Kasi      2026-01-15 40) public User getUser(String id) {
abc1234 (Kasi      2026-01-15 41)     if (id == null) {
def5678 (Sonali    2026-01-20 42)         return null; // Bug here!
abc1234 (Kasi      2026-01-15 43)     }
abc1234 (Kasi      2026-01-15 44)     return userRepo.findById(id);

Now you know Sonali changed line 42 on Jan 20. You can ask her about it or check the commit.

2
Blame Specific Lines
Terminal 
git blame -L 40,50 src/UserService.java
3
See File's Complete History
Terminal 
git log -p -- src/UserService.java
-p shows the actual diff in each commit. -- separates options from file path.
üí°
git blame isn't about blaming people‚Äîit's about understanding code history and context!

5.2 Context Switching with Stash

Output:

Your stashed changes are back! Continue where you left off.

Fetch vs Pull
git fetch

Downloads changes but doesn't merge

Safe to run anytime - lets you review first

git pull

Downloads and merges automatically

Equivalent to: git fetch + git merge

Safe approach: Fetch first, then review

git merge origin/main                 # Merge when ready

6.2 Resolving Merge Conflicts

Intermediate

When two developers change the same lines - how to fix it

üé¨ The Situation

You and a teammate both modified the same function. When you try to merge or pull, Git says there's a conflict!

üîç Anatomy of a Conflict

Git marks conflicts with special markers in the file:

public String getUserName() {
<<<<<<< HEAD
    return firstName + " " + lastName;
=======
    return lastName + ", " + firstName;
>>>>>>> feature/name-format
}

<<<<<<< HEAD = Your current changes

======= = Divider

>>>>>>> feature/name-format = Incoming changes

üìù Resolving Steps

1
See Conflicting Files
Terminal 
git status
Output: 
Unmerged paths:
  both modified: src/UserService.java
2
Edit File - Choose What to Keep

Open the file and decide which version (or combine both):

// After resolving - keep what makes sense:
public String getUserName() {
    return lastName + ", " + firstName; // Kept incoming format
}

Remove ALL conflict markers (<<<, ===, >>>)

3
Mark as Resolved & Commit
Terminal 
git add src/UserService.java
git commit -m "Resolve merge conflict in UserService"
üí°
VS Code has excellent conflict resolution UI - it shows "Accept Current", "Accept Incoming", "Accept Both" buttons!

7.1 Undoing Commits

Advanced

Undo local commits, amend messages, and safely undo published commits

üìù Common Undo Scenarios

1
Fix Last Commit Message
Terminal 
git commit --amend -m "INF-5678: Correct commit message"
‚ö†Ô∏è
Only amend commits that haven't been pushed! Amending rewrites history.
2
Undo Last Commit (Keep Changes)
Terminal 
# Keep changes staged
git reset --soft HEAD~1

# Keep changes unstaged
git reset HEAD~1
3
Undo a Published Commit (Safe Way)

Use revert to create a new commit that undoes the changes:

Terminal 
git revert abc1234
git revert is safe for shared branches because it doesn't rewrite history.

7.2 Recovering Lost Work with Reflog

Advanced

The "time machine" that can recover almost anything

üé¨ The Situation

Panic! You ran git reset --hard and lost commits. Or you deleted a branch with work on it. Is it gone forever?

No! Git's reflog remembers everything for ~30 days.

üìù Recovery Steps

1
View Reflog (Git's Activity Log)
Terminal 
git reflog
Output: 
abc1234 HEAD@{0}: reset: moving to HEAD~3
def5678 HEAD@{1}: commit: INF-5678: Important feature  ‚Üê Your lost commit!
ghi9012 HEAD@{2}: commit: INF-5677: Another commit
...
2
Recover to That Point
Terminal 
git reset --hard def5678

Your commits are back!

‚úÖ
Reflog is your safety net. As long as you committed something, it can usually be recovered.
